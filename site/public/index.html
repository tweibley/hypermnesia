hy·per·mne·sia n.

unusually vivid or complete memory; abnormally sharp recall of the past.

Your coding agent forgets.
This is its memory.

Hypermnesia is a local-first memory system for Claude Code, Cursor, and Google Antigravity. It captures the decisions and conventions behind your codebase from real sessions, retires them when they go stale, and injects the survivors into new ones — on-device, no telemetry, and measured: automatic hydration took a memoryless agent from 3/24 to 15/24 on tasks whose answer existed only in memory.

01 / The problem

Sessions end. Context dies.

Yesterday you told your agent the API speaks cents, not dollars. Today it suggests the float again. Every session re-litigates settled decisions, re-suggests banned patterns, and loses the why behind the code when the context window closes. That tax is paid per session, forever — unless something remembers.

02 / The artifact

Not a blob of chat history. Every capture is a typed record with scope rules, evidence counters, a decay state, and provenance it can be audited by.

The type decides how a memory behaves: conventions carry scope rules like the one above, concerns carry severity, decisions carry the rejected alternative — and code refs pin to commits and deep-link into your editor or GitHub.

03 / How it works

The loop

  1. Capture

    Hooks checkpoint the transcript during the session — capture is incremental and asynchronous, so nothing ever blocks your agent.

  2. Classify

    A pluggable LLM (Gemini by default, claude -p fallback) reads each new slice and extracts typed memories.

  3. Curate

    Confident captures go live on their own; risky writes wait as drafts for your confirm. Near-duplicates merge, and a newer memory that contradicts an older one retires it.

  4. Decay

    Memories lose confidence as they age unless use revalidates them. Staleness isn’t silent — it’s a visible, first-class state.

    freshagingstaledormant
  5. Hydrate

    The relevant, still-trustworthy memories are injected into new sessions at start, and per-prompt by relevance — a small, bounded, confidence-gated set.

↺ back to 01 — a hydrated session is itself captured; the loop closes. And overnight, a slower loop runs over the week: 04 / Dreams.

04 / Overnight

It dreams — with receipts.

05 / Privacy

Local-first, verifiably

  • store one SQLite file — ~/Library/Application Support/Hypermnesia/memory.db
  • telemetry none. No accounts, no cloud store.
  • leaves your machine exactly two things, both to the classifier you configure: transcript slices for classification, and stored memory summaries plus your question for ask. That’s the whole list — the threat model is in SECURITY.md ↗.
  • this site self-hosted fonts; one third-party request — privacy-friendly PostHog page analytics (no memory content, Do Not Track respected). The app itself still sends nothing.

08 / Install

Quick start

⬇  Download for macOS Universal binary · signed & notarized · macOS 14+

Unzip, drag to Applications, open — the app walks you through setup, no terminal required (the CLI ships inside the app). Prefer the terminal? The bundled CLI does the same work headless:

alias hypermnesia=/Applications/Hypermnesia.app/Contents/Resources/hypermnesia
hypermnesia backfill --project ~/some/repo     # seed memory from past sessions
hypermnesia setup --with-mcp                   # hooks + recall/ask/remember
open -a Hypermnesia

Requires macOS 14+ and claude on PATH. A GEMINI_API_KEY enables the recommended classifier; without one it falls back to claude -p. hypermnesia doctor verifies the result. Prefer building from source? The README has you covered.

09 / FAQ

Fair questions

Why not just CLAUDE.md?

Hypermnesia writes your CLAUDE.md — hypermnesia export --markdown renders a project’s confirmed memory as a committable MEMORY.md, sections, provenance, and all. The difference is everything around that file: memories decay unless they keep proving useful, carry applied/overridden evidence counters, are retired by newer contradicting captures, and are injected by relevance instead of as one ever-growing preamble. A hand-tended CLAUDE.md goes stale silently; here staleness is a first-class, visible state.

Isn’t this just RAG?

The retrieval part is deliberately boring (FTS5 + embeddings). The interesting parts sit around it: capture is automatic from real sessions; memories are typed, with structure like “does NOT apply to”; contradictions are detected at capture time; and decay + belief decide whether a memory is still trustworthy, not just similar. The evals benchmark against a paste-the-fact-into-the-prompt oracle — on explicit-memory tasks, hydration matched it.

Does injected memory ever hurt?

Measured: yes, once — memory injection broke 1 of 18 adversarial control tasks that the baseline passed (details, including the failure, in evals/). The mitigations are structural: overridden memories lose belief, stale ones fall below the injection floor until revalidated, and risky writes land as drafts you confirm before they hydrate anything.

Does it slow sessions down or eat tokens?

Hydration injects a small, bounded set: confirmed memories above a confidence floor at session start, and up to 8 relevance-ranked ones per prompt. Capture is asynchronous — hooks checkpoint and return; classification runs out-of-band, so nothing blocks the session. The only recurring spend is classification, on the model you choose.

Dreams run an LLM over my sessions while I sleep?

Only if you turn them on — Dreams are off by default, and enabling shows the cost estimate first. The budget is one classifier call per project per night, under a nightly cap you set; it runs only while the machine is idle on adequate power, and stops the moment you’re back. Every journal entry records its calls and estimated cost. Dreams never confirm memories or install skills on their own: drafts go through the same review inbox as everything else, and skills stage for an explicit install — with a diff before any update, one-tap uninstall, and honest report-backs on whether they were ever used. A night with nothing real to say is logged as quiet, never padded.

Why macOS-only? Will there be a Linux build?

The engine and CLI are UI-free Swift over GRDB/SQLite — nothing in the capture ↔ hydrate loop is macOS-specific; the SwiftUI app is. A headless Linux build of the CLI + hooks is plausible; it just isn’t built or tested yet. If that’s what’s blocking you, open an issue.

10 / Inventory

The complete list, for the thorough

Everything above is the argument. This is the manifest — every shipped capability, one line each.

Typed memories

Seven types with scope rules (“does NOT apply to”), evidence counters, provenance pinned to commits.

Decay + belief

Confidence = belief × freshness; staleness is a visible state, and overridden memories lose trust structurally.

Contradiction handling

Detected at capture, confirm-gated supersede, automatic newer-wins sweep for confirmed pairs, Restore as undo.

Review inbox

Risky writes land as drafts; keyboard triage, bulk confirm/dismiss, every action undoable for 8 seconds.

Automatic capture

Incremental, asynchronous hooks for Claude Code, Cursor, and Google Antigravity — nothing blocks a session.

Backfill

Seed memory from the sessions you’ve already run, consent dialog first — count and cost before anything burns.

Hydration + MCP

Bounded, relevance-ranked injection at session start and per prompt; recall/ask/remember from any MCP client.

Memory Dreams

Idle-after-wake consolidation: evidence-backed epiphanies, draft memories, skill proposals, one morning digest.

Skill lifecycle

Staged SKILL.md preview, install to detected layouts, update-with-diff, one-tap uninstall, usage report-backs.

Brain MRI

Live replay of real session events over the memory anatomy — plus a night-palette REM replay after a dream.

Five more views

Ask

Reality-check audit
